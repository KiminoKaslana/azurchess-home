# 证书申请

Nginx 端口转发，使用 acme.sh 申请证书，使用 Google Cloud 的 External Account Key 进行认证。

## 安装 acme.sh
```bash
curl https://get.acme.sh | sh -s email=my@example.com
```

## 创建 External Account Key
```bash
gcloud publicca external-account-keys create
```

External Account Key 信息如下：
```bash
[b64MacKey: <YOUR_B64MACKEY>
keyId: <YOUR_KEY_ID>]
```

## ACME 注册 Google Cloud 账号
```bash
~/.acme.sh/acme.sh --register-account \
  --server google \
  --eab-kid "<YOUR_KEY_ID>" \
  --eab-hmac-key "<YOUR_B64MACKEY>"
```

## 申请证书

需要 root 权限运行。下面会申请一张多域名证书，同时覆盖：

```bash
sudo su

~/.acme.sh/acme.sh --issue --nginx \
  --server google \
  -d azurchess.YOUR_DOMAIN.com \
  -d user.azurchess.YOUR_DOMAIN.com \
  -d game.azurchess.YOUR_DOMAIN.com
```

`--issue --nginx` 只负责临时改 Nginx 完成 HTTP-01 验证并签发证书；签发成功后会恢复 Nginx 配置，不会自动长期启用 HTTPS。

## 安装证书到 Nginx

签发成功后，将证书安装到固定路径，供 `/etc/nginx/conf.d/azurchesscloud.conf` 引用：

```bash
mkdir -p /etc/nginx/ssl/azurchess

~/.acme.sh/acme.sh --install-cert \
  -d azurchess.YOUR_DOMAIN.com \
  -d user.azurchess.YOUR_DOMAIN.com \
  -d game.azurchess.YOUR_DOMAIN.com \
  --key-file /etc/nginx/ssl/azurchess/privkey.pem \
  --fullchain-file /etc/nginx/ssl/azurchess/fullchain.pem \
  --reloadcmd "systemctl reload nginx"
```

注意：`--install-cert -d` 必须使用签发时的第一个 `-d`，也就是 `azurchess.YOUR_DOMAIN.com`。

## Nginx 配置

`/etc/nginx/conf.d/azurchesscloud.conf` 中三个 HTTPS server 共用同一组证书文件：

```nginx
ssl_certificate /etc/nginx/ssl/azurchess/fullchain.pem;
ssl_certificate_key /etc/nginx/ssl/azurchess/privkey.pem;
```

修改配置或安装证书后检查并重载：

```bash
nginx -t && systemctl reload nginx
```
